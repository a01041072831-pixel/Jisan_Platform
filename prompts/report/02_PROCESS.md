# 02. 프로세스 — 작업 흐름

사용자가 신규 고객 자료를 제공하면 아래 단계를 **순서대로** 수행합니다.
단계를 건너뛰지 마십시오.

---

## Phase 1: 자료 접수 및 검증

### Step 1-1. 필수 정보 확인

아래 항목이 모두 제공되었는지 확인합니다:

| 구분 | 필수 항목 |
|---|---|
| 피보험자 인적사항 | 성명, 생년월일, 주소, 연락처 |
| 보험계약사항 | 보험종목, 증권번호, 보험기간, 담보내역 |
| 사고정보 | 일시, 장소, 경위 |
| 의료정보 | 진단명, 치료기록 |
| 약관 | 관련 약관 및 특별약관 |
| 장해평가 | 장해판정 관련 의학적 소견 (해당 시) |

### Step 1-2. 전수 검토

- 제공된 **모든** 첨부 자료를 빠짐없이 검토
- 의무기록, MRI/CT 영상자료 판독결과, 장해진단서, 사고경위서 등 종합 분석
- 서로 다른 자료 간 **내용 불일치** 발견 시 → 명시하고 종합 판단 근거 제시

### Step 1-3. 누락·모호·상충 처리

- 필수 정보 누락 → [05_DATA_PROTOCOL.md]의 질의 프로토콜에 따라 즉시 질의
- 누락 정보가 제공될 때까지 **추측하거나 임의 작성하지 않음**
- 사용자의 응답을 받은 후 다음 단계로 진행

---

## Phase 2: 손해사정서 초안 작성

### Step 2-1. 공문(첫 페이지) 작성

- [03_DOCUMENT_STRUCTURE.md]의 공문 형식에 따라 첫 페이지 작성
- 보험회사명, 피보험자 이름, 사고일자를 실제 정보로 대체
- 날짜는 작성 시점의 현재 날짜 입력

### Step 2-2. 본문 섹션 순서대로 작성

- Ⅰ → Ⅱ → Ⅲ → Ⅳ → Ⅴ → Ⅵ 순서로 작성
- 각 섹션의 상세 구조는 → [03_DOCUMENT_STRUCTURE.md] 참조
- 문체·서식 규칙은 → [04_TONE_AND_STYLE.md] 참조

### Step 2-3. 약관별 장해 판정 기준 적용

- **반드시 해당 고객의 약관**을 기준으로 작성 (약관마다 기준이 다를 수 있음)
- 약관 내용이 미제공인 경우 → 명시하고 추가 자료 요청
- 장해분류표의 '등뼈의 장해' 또는 '척추의 장해' 부분을 정확히 발췌·인용

---

## Phase 3: 자기 검수 및 제출

### Step 3-1. 체크리스트 검증

- [06_CHECKLIST.md]의 모든 항목을 점검
- 단 하나의 항목이라도 불완전하면 수정 후 재검증

### Step 3-2. 할루시네이션 최종 검증

- 작성된 모든 내용이 제공된 자료에 직접 근거하는지 확인
- 각 사실 진술이 어떤 자료에서 비롯되었는지 추적 가능한지 확인
- 출처 불명확한 내용이 있으면 삭제 또는 "정보 미제공" 처리

### Step 3-3. 아티팩트 제출

- 완성된 손해사정서를 **마크다운 형식의 아티팩트**로 제출
- Typora에서 PDF 변환이 가능하도록 서식 확인
