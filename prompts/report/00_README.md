# 손해사정서 작성 봇 — MD 파일 구조 안내

## 파일 구성

| 파일명 | 역할 | 핵심 질문 |
|---|---|---|
| `01_SYSTEM.md` | 봇의 정체성·역할·핵심 원칙 | "너는 누구이고, 무엇을 지켜야 하는가?" |
| `02_PROCESS.md` | 작업 흐름 (입력→검증→작성→검수) | "어떤 순서로 일하는가?" |
| `03_DOCUMENT_STRUCTURE.md` | 손해사정서 섹션별 작성 구조 | "결과물은 어떤 형태인가?" |
| `04_TONE_AND_STYLE.md` | 문체·어조·마크다운 서식 규칙 | "어떤 말투와 포맷을 쓰는가?" |
| `05_DATA_PROTOCOL.md` | 데이터 검증·누락 처리·할루시네이션 방지 | "모르는 건 어떻게 처리하는가?" |
| `06_CHECKLIST.md` | 최종 품질 검증 체크리스트 | "제출 전 무엇을 확인하는가?" |

## 설계 원칙

1. **한 파일 = 한 관심사**: 역할, 프로세스, 구조, 문체, 데이터, 검수를 분리하여 각각 독립적으로 수정 가능
2. **참조 관계 명시**: 파일 간 참조가 필요한 곳은 `→ [파일명] 참조`로 표기
3. **우선순위 부여**: 각 파일 내에서 MUST / SHOULD / MAY로 중요도 구분
4. **예시 포함**: 추상적 규칙만이 아니라 구체적 예시를 포함하여 클로드의 이해도 향상

## 사용법

- **클로드 프로젝트**: 6개 파일 전체를 Project Knowledge에 업로드
- **클로드 코드**: 프로젝트 루트의 `.claude/` 디렉토리에 배치하거나 `CLAUDE.md`에서 참조
