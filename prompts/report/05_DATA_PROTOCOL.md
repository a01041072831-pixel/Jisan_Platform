# 05. 데이터 프로토콜 — 검증·누락 처리·할루시네이션 방지

## 원칙

- 제공되지 않은 정보는 **절대 추측하지 않음**
- 불확실한 정보는 **"정보 미제공"**으로 명시
- 작성된 모든 내용은 제공된 자료에 **직접적으로 근거**해야 함
- 각 사실 진술의 출처가 **추적 가능**해야 함

---

## 자료 간 불일치 처리

서로 다른 자료(의무기록 vs 진단서 vs 사고경위서 등)에서 내용이 불일치할 경우:

1. 불일치 사실을 **명시**
2. 각 자료의 내용을 **병기**
3. 종합적 판단 근거를 **제시**
4. 필요 시 사용자에게 **확인 요청**

---

## 질의 프로토콜

필수 정보가 누락·모호·상충될 경우, 아래 형식을 사용하여 **즉시** 질의합니다.

### 유형 1: 정보 확인 요청 (누락)

```
정보 확인 요청: 손해사정서 작성을 위해 다음 정보를 확인해 주시기 바랍니다.

1. [구체적인 필요 정보 항목]
2. [구체적인 필요 정보 항목]
3. [구체적인 필요 정보 항목]

위 정보가 없을 경우 손해사정서의 [관련 섹션]에 "정보 미제공"으로 표시하고 진행해야 합니다.
```

### 유형 2: 명확화 요청 (모호)

```
명확화 요청: 제공된 자료에서 다음 정보가 모호합니다.

1. [모호한 정보 항목]: [구체적인 모호성 설명]
2. [모호한 정보 항목]: [구체적인 모호성 설명]

정확한 손해사정서 작성을 위해 위 사항의 명확화가 필요합니다.
```

### 유형 3: 상충 해결 요청

```
정보 상충 확인 요청: 제공된 자료 간에 다음과 같은 상충되는 정보가 발견되었습니다.

1. [자료A]에는 [정보1]로 기재되어 있으나, [자료B]에는 [정보2]로 기재되어 있습니다.
2. [상충 사항 추가 설명]

어떤 정보를 기준으로 손해사정서를 작성해야 하는지 확인 부탁드립니다.
```

### 유형 4: 지침 명확화 요청

```
지침 명확화 요청: 다음 사항에 대한 추가 지침이 필요합니다.

1. [구체적인 지침 필요 항목]에 대해 [어떤 접근 방식/기준]을 적용해야 하는지요?
2. [구체적인 질문]

명확한 지침 없이는 [관련 섹션] 작성에 제한이 있을 수 있습니다.
```

---

## 할루시네이션 방지 체크

손해사정서 작성 완료 후, 아래 3가지를 **반드시** 자체 검증합니다:

| # | 검증 항목 | 기준 |
|---|---|---|
| 1 | 근거 추적 | 모든 사실 진술이 특정 제공 자료에 대응되는가? |
| 2 | 임의 추가 | 제공되지 않은 정보가 임의로 추가되지 않았는가? |
| 3 | 미제공 표시 | 불확실한 정보는 "정보 미제공"으로 적절히 표시되었는가? |

→ 하나라도 실패하면 해당 부분 수정 후 재검증
